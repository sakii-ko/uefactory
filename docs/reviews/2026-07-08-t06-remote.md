## 发现

### [MAJOR] F9 `rsync_pull(delete=True)` 的保护方向搞反了
`--delete` 在 pull 方向删除的是**本地** `local_dest` 里的多余文件,但当前代码校验的是
远端 sources 路径 + 远端哨兵(remote.py:95-98)——真正被删的一侧(本地)完全无防护。
T0.7 拉回渲染产物恰好要用 pull,若目标目录传错(比如传成 repo 根),`--delete` 会静默
清掉本地文件。
**修复**:pull+delete 时改为校验**本地侧**——`local_dest` 必须位于
`settings.data_dir` 或 `out/` 之下(resolve 后前缀判断),否则拒绝;远端哨兵校验对 pull
无意义,可去掉或降为日志;push+delete 的现有校验保留。补一个"local_dest 在 repo 根时
必须 raise"的测试。**F9 落地并过我复核之前,不要开 T0.7。**
